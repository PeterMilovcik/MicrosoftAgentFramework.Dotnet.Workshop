# Bonus Module — RPG Game Master

> **An AI-powered text RPG where an LLM Game Master orchestrates dynamic agents, generates worlds procedurally, and creates NPC personalities on-the-fly — all using the Magentic-One pattern from Module 09.**

---

## What This Module Demonstrates

This bonus module repurposes the **manager-driven orchestration pattern** (Magentic-One) from Module 09 to build a fully playable text-based RPG. It showcases several advanced agent patterns beyond the core workshop:

| Pattern | How It's Used |
|---------|---------------|
| **Dynamic agent creation** | NPC agents are created at runtime from LLM-generated system prompts — agents creating agents |
| **Procedural content generation** | Locations, NPCs, creatures, and items are generated by specialized sub-agents on demand |
| **Persistent world state** | Generated content is saved as JSON files and reloaded across sessions (save/load game) |
| **Tool-mediated persistence** | Each sub-agent uses tools (`SaveLocation`, `SaveNPC`, etc.) to persist its output |
| **Multi-tier orchestration** | Outer player-turn loop → inner GM routing loop → sub-workflows (combat, dialogue) |
| **Stat-based combat with dice tools** | Combat Narrator agent uses `RollDice` tool for random outcomes, not fabricated numbers |

## Gameplay

1. **Create a character** — choose a name and a world theme (Dark Fantasy, Enchanted Forest, Sci-Fi Station, etc.)
2. **Explore** — the GM generates your starting location with NPCs. Move through exits to discover new areas.
3. **Talk to NPCs** — each NPC has a unique personality agent created on-the-fly. They may offer quests.
4. **Fight creatures** — dice-based combat with attack/defend/flee/use-item options.
5. **Level up** — gain XP from defeated creatures. Every level-up increases your stats.
6. **Save and resume** — your world persists in JSON files under `game-data/`.

## Key Concepts: Module 09 → RPG

| Module 09 Concept | RPG Adaptation |
|-------------------|---------------|
| Magentic Manager | Game Master — routes to specialist agents per player turn |
| `ManagerDecision` JSON | `GameMasterDecision` (routing) + `PlayerPresentation` (narrative + options) |
| Researcher agent (with tools) | World Architect, NPC Weaver, Creature Forger (each with persistence tools) |
| Diagnostician / Critic | Combat Narrator (with dice + stat tools) |
| Scribe → TriageCard | GM → PlayerPresentation (structured output for player) |
| Shared history | Game state summary + game log (trimmed to last 20 entries) |
| Stateless per-turn sessions | Same — fresh `AgentSession` per agent call |
| Confidence threshold → DONE | `PRESENT_TO_PLAYER` signal when generation is complete |
| Max 8 iterations | Max 6 inner iterations per player turn |

## Token Usage Notes

This module uses **significantly more tokens** than other modules due to:
- Multiple LLM calls per player turn (GM routing + sub-agent generation + GM presentation)
- Location generation turns are the most expensive (~4-6 LLM calls)
- Dialogue and combat involve additional LLM calls per round
- **Estimated**: ~2,000-5,000 tokens per player turn; ~15,000-30,000 for a 10-turn session

The token tracker displays a summary at session exit so you can monitor usage.

## Relation to the Workshop

This module is **completely self-contained** — it has its own assets folder, its own prompts, and its own game data directory. It does not share any files with the core workshop modules (00-09).

It builds on concepts from:
- **Module 02** — Tool registration via `AIFunctionFactory.Create()`
- **Module 04** — Multi-step workflows with separate agent sessions
- **Module 05** — Human-in-the-loop (player choices as approval gates)
- **Module 09** — Manager-driven orchestration (the core pattern)

And adds new patterns not covered elsewhere:
- **Dynamic agent creation** from LLM-generated instructions
- **Procedural content generation** with tool-based persistence
- **Nested sub-workflows** (combat and dialogue within the main game loop)
